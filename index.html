Object Diagram

Object Diagram merupakan suatu rancangan sistem yang dipakai mengambarkan tentang nama objek, atribut, dan metode yang digunakan. Object Diagram menghasilkan gambaran struktur model sebuah sistem, dalam kurun waktu tertentu. Diagram objek yang berasal dari class diagram, sehingga object diagram tergantung pada class diagram.

Fungsi Object Diagram

Menggambarkan seperti apa hubungan antara objek-objek yang ada

Melakukan pengujian terhadap Class Diagram dengan memanfaatkan Object Diagram tertentu

Memeriksa literasi pada sebuah program

Menghasilkan gambar dengan resolusi tinggi dari sebuah sistem yang sedang dikembangkan

Membuat sebuah prototype

Membuat pemodelan data yang bersifat kompleks

Memahami sebuah sistem melalui perspektif yang lebih praktis

Diagram dan Penjelasan Komponen

Mempresentasikan objek dalam sistem. Di dalamnya berisi nama object, nama class, atribut dan value.

Menggambarkan relasi antara satu object dengan object yang lain

Contoh Skenario

Pada skenario disamping terdapat sistem pemesanan roti yang memiliki tiga class utama yaitu:
  • Customer
  • VarianRoti
  • Pemesanan
Penjelasan Diagram

Objek Customer, yaitu Athena, memiliki relasi dengan Pemesanan dimana Athena akan melakukan pemesanan. Objek Pemesanan memiliki relasi dengan VarianRoti ini menunjukkan pemesanan yang dilakukan oleh Athena, yaitu roti varian Madeleine. Sedangkan untuk objek Customer tidak memiliki relasi secara langsung dengan VarianRoti, tetapi melalui Pemesanan relasi tersebut diimplikasikan
